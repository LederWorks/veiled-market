# Architecture: Commons Build — Dual-Platform Plugin Generation

## Executive Summary

Each plugin's human-authored content lives in a single `plugins/<name>/commons/` source tree. A new `scripts/build-platforms.py` script reads that tree and writes two derived output directories — `plugins/<name>/claude/` and `plugins/<name>/copilot/` — applying the exact file-level transforms each platform requires (SKILL.md vs workflow.md for hidden skills, `<name>.md` vs `<name>.agent.md` for agents). The `finalize.py` pipeline gains a `build` step that runs immediately after `promote`, and `_write_marketplace_files()` is updated to route each generated marketplace file to the correct platform subdirectory using a new `platform_sources` registry field in `sources/plugins.json`.

---

## Three Options

| | **Option A — `commons/` subdirectory** ✅ | **Option B — Extended `sources/plugins.json`** | **Option C — Jinja2 Templates** |
|---|---|---|---|
| **Single source of truth** | `commons/` per plugin | `sources/plugins.json` + flat plugin dir | Template files in `commons/` |
| **File-level transforms** | `build-platforms.py` copies + renames | `_write_marketplace_files()` injects platform-specific paths | Jinja2 renderer |
| **SKILL.md → workflow.md** | Done by build script | Not possible (files unaffected) | Done by renderer |
| **Agent renaming** | Done by build script | Not possible (files unaffected) | Done by renderer |
| **Backward compat** | `commons/` absence = skip build gracefully | Fully backward compat | Requires full migration |
| **stdlib only** | ✅ | ✅ | ❌ Jinja2 is not stdlib |
| **Complexity** | Medium (new script + 2 new step) | Low (schema extension only) | High (template language + tooling) |
| **Inspectable outputs** | Explicit files on disk per platform | Implicit — no files, just paths | Explicit rendered files |

**Recommendation: Option A.** Option B cannot perform the required file-level transforms (renaming SKILL.md to workflow.md for hidden skills; renaming agent files) because `_write_marketplace_files()` only generates manifests, not file content. Option C violates the stdlib-only constraint. Option A produces explicit, auditable per-platform directories that exactly mirror what each platform's installer receives, follows the `compose.py` precedent (filter → copy → write manifest), and degrades gracefully for legacy plugins that have no `commons/`.

## Component Design

```
│   │   └── SKILL.md          # Frontmatter: user-invocable: true (or absent = visible)
│   ├── terraform-apply/
│   │   └── SKILL.md          # Frontmatter: user-invocable: false (hidden skill)
│   ├── terraform-destroy/
│   │   └── SKILL.md
│   ├── terraform-format/
│   │   └── SKILL.md
│   ├── terraform-import/
│   │   └── SKILL.md
```

**Generated outputs (committed, not .gitignored):**

```
plugins/terraformer/
├── commons/                  # Source of truth (human-authored)
├── claude/                   # Generated by build-platforms.py
│   ├── plugin.json
│   ├── agents/
│   │   └── terraformer.md    # Kept as .md (Claude Code prefers this)
│   ├── skills/
│   │   ├── terraform-plan/SKILL.md
│   │   ├── terraform-apply/SKILL.md   # user-invocable: false stays in frontmatter
│   │   └── ...
│   ├── hooks.json
│   └── .mcp.json
└── copilot/                  # Generated by build-platforms.py
    ├── plugin.json
    ├── agents/
    │   └── terraformer.agent.md   # Renamed: .md → .agent.md
    ├── skills/
    │   ├── terraform-plan/SKILL.md
    │   ├── terraform-apply/workflow.md  # Hidden skill: SKILL.md → workflow.md, flag stripped
    │   └── ...
    ├── hooks.json
    └── .mcp.json
```

> **Why commit generated outputs?** The marketplace installers fetch files directly from the repo tree. Generated files must be present in the committed tree; they cannot be generated at install time. This mirrors the existing convention where `plugin.json`, `agents/`, and `skills/` are all committed artifacts.

**Purpose**: JSON Schema for `commons/plugin.base.json`; defines the platform-neutral plugin manifest that is the input to the build.

**Interface**:

```json
  "description": "Platform-neutral source manifest living in plugins/<name>/commons/plugin.base.json. build-platforms.py reads this and generates platform-specific plugin.json files.",
  "type": "object",

  "additionalProperties": false,
  "properties": {
    "$schema":     { "type": "string" },

    "keywords":   { "type": "array", "items": { "type": "string" } },
    "category":   { "type": "string" },
    "tags":       { "type": "array", "items": { "type": "string" } },
    "categories": {
      "type": "array",
      "uniqueItems": true,
      "items": {
        "type": "string",
        "enum": ["AI Agents","Coding","Data","DevOps","Marketing","Media","Other","Productivity","Security","Writing"]
      }
    },
    "agents":     { "type": "string", "description": "Relative path to agents/ directory. Always 'agents/'." },
    "skills":     { "type": "string", "description": "Relative path to skills/ directory. Always 'skills/'." },
    "hooks":      { "type": "string", "description": "Relative path to hooks.json. Always 'hooks.json'." },
    "mcpServers": { "type": "string", "description": "Relative path to .mcp.json. Always '.mcp.json'." }
  }
}
```

**Note on `user-invocable`**: This field lives exclusively in `SKILL.md` frontmatter as `user-invocable: false`. It is NOT a `plugin.base.json` field. The build script reads it from each skill's frontmatter to determine the hidden-skill transform for Copilot.

**Note on `commands` with `${CLAUDE_PLUGIN_ROOT}`**: When a future plugin needs Claude-specific command paths, add them to `plugin.base.json` as a `commands` array. `build_platforms.py` will inject `${CLAUDE_PLUGIN_ROOT}/` prefixes for the Claude output and strip them for the Copilot output. Terraformer does not use `commands`, so this path is unexercised in the initial implementation.

**Dependencies**: None (standalone schema).

### `scripts/build-platforms.py`

**Purpose**: Reads `plugins/<name>/commons/` and writes `plugins/<name>/claude/` and `plugins/<name>/copilot/`, applying all platform-specific file transforms. Callable as a CLI tool and importable by `finalize.py`.

**Interface**:
```python
# CLI
# python3 scripts/build-platforms.py --plugin terraformer [--dry-run] [--commons-dir path]

def build_plugin(
    plugin: str,
    commons_dir: pathlib.Path | None = None,
    dry_run: bool = False,
) -> None:
    """Entry point called by finalize.py and the CLI.

    Derives commons_dir as plugins/<plugin>/commons/ when None.
    Writes plugins/<plugin>/claude/ and plugins/<plugin>/copilot/.
    Raises SystemExit(1) on any error.
    """

def _build_claude(commons_dir: pathlib.Path, out: pathlib.Path, dry_run: bool) -> None:
    """Transform commons/ → claude/ output directory."""

def _build_copilot(commons_dir: pathlib.Path, out: pathlib.Path, dry_run: bool) -> None:
    """Transform commons/ → copilot/ output directory."""

def _copy_skills_claude(src: pathlib.Path, dst: pathlib.Path) -> None:
    """Copy skills/ as-is; preserve user-invocable: false in SKILL.md frontmatter."""

def _copy_skills_copilot(src: pathlib.Path, dst: pathlib.Path) -> None:
    """Copy skills/, renaming SKILL.md → workflow.md for hidden skills and stripping the flag."""

def _copy_agents_claude(src: pathlib.Path, dst: pathlib.Path) -> None:
    """Copy agents/<name>.md → out/agents/<name>.md unchanged."""

def _copy_agents_copilot(src: pathlib.Path, dst: pathlib.Path) -> None:
    """Copy agents/<name>.md → out/agents/<name>.agent.md (insert .agent before .md)."""

def _write_platform_manifest(
    base: dict,
    out_dir: pathlib.Path,
    platform: str,          # "claude" | "copilot"
) -> None:
    """Generate plugin.json for the given platform from plugin.base.json fields."""

# Reused from compose.py (copy-imported, not imported, to keep build-platforms.py standalone)
def _parse_frontmatter(content: str) -> dict: ...
def _body_after_frontmatter(content: str) -> str: ...
def _render_frontmatter(fm: dict) -> str: ...
```

**Full Algorithm**:

```
build_plugin(plugin, commons_dir=None, dry_run=False):
  1. Resolve commons_dir = plugins/<plugin>/commons/ (or CLI override)
  2. Validate commons_dir exists and commons_dir/plugin.base.json exists → exit 1 on failure
  3. Load base = json.load(commons_dir / "plugin.base.json")
  4. For each platform in ["claude", "copilot"]:
       out_dir = plugins/<plugin>/<platform>/
       if not dry_run:
           shutil.rmtree(out_dir, ignore_errors=True)   # clean slate
           out_dir.mkdir(parents=True)
       _build_<platform>(commons_dir, out_dir, dry_run)

_build_claude(commons_dir, out, dry_run):
  1. Copy commons/skills/ → out/skills/
       For each skill_dir in commons/skills/:
           shutil.copytree(skill_dir, out/skills/<name>)
       (SKILL.md files are copied verbatim; user-invocable: false in frontmatter
        is correct for Claude Code and requires no transformation)
  2. Copy commons/agents/ → out/agents/
       For each <name>.md in commons/agents/:
           shutil.copy2(src, out/agents/<name>.md)     # name unchanged
  3. Copy commons/hooks.json → out/hooks.json  (if exists)
  4. Copy commons/.mcp.json  → out/.mcp.json   (if exists)
  5. _write_platform_manifest(base, out, "claude")

_build_copilot(commons_dir, out, dry_run):
  1. For each skill_dir in commons/skills/:
       skill_md = skill_dir / "SKILL.md"
       content = skill_md.read_text()
       fm = _parse_frontmatter(content)
       body = _body_after_frontmatter(content)
       out_skill = out/skills/<name>/
       copy all non-SKILL.md files in skill_dir → out_skill (shutil.copytree ignore SKILL.md)
       if fm.get("user-invocable") == "false" or fm.get("user-invocable") is False:
           # Hidden skill: rename to workflow.md and strip the flag
           fm.pop("user-invocable", None)
           (out_skill / "workflow.md").write_text(_render_frontmatter(fm) + body)
       else:
           # Normal skill: copy as SKILL.md
           shutil.copy2(skill_md, out_skill / "SKILL.md")
  2. For each <name>.md in commons/agents/:
       shutil.copy2(src, out/agents/<name>.agent.md)   # insert .agent before .md
  3. Copy commons/hooks.json → out/hooks.json (if exists)
  4. Copy commons/.mcp.json  → out/.mcp.json  (if exists)
  5. _write_platform_manifest(base, out, "copilot")

_write_platform_manifest(base, out_dir, platform):
  manifest = {k: v for k, v in base.items() if not k.startswith("$")}
  # Paths are identical on both platforms (agents/, skills/, hooks.json, .mcp.json)
  # because the actual file differences are handled in the directory, not the manifest
  #
  # Future: if base contains "commands" with ${CLAUDE_PLUGIN_ROOT}:
  #   if platform == "copilot": strip ${CLAUDE_PLUGIN_ROOT}/ prefix from each command path
  #   if platform == "claude":  keep as-is (Claude Code resolves the var at runtime)
  (out_dir / "plugin.json").write_text(json.dumps(manifest, indent=2) + "\n")

_render_frontmatter(fm):
  lines = ["---"]
  for key, val in fm.items():
      if isinstance(val, list):
          items_str = ", ".join(str(v) for v in val)
          lines.append(f"{key}: [{items_str}]")
      elif isinstance(val, bool):
          lines.append(f"{key}: {'true' if val else 'false'}")
      else:
          lines.append(f"{key}: {val}")
  lines.append("---")
  return "\n".join(lines) + "\n"
```

**Error handling**:

| Condition | Behavior |
|-----------|----------|
| `commons/` directory missing | Print `[build] No commons/ for <plugin>, skipping.` and return (non-fatal; supports legacy flat layout) |
| `commons/plugin.base.json` missing | `sys.exit(1)` with clear error |
| `commons/skills/` missing | Skip skill copy; log warning |
| `commons/agents/` missing | Skip agent copy; log warning |
| `commons/hooks.json` missing | Skip; log info (optional file) |
| `commons/.mcp.json` missing | Skip; log info (optional file) |
| Skill directory has no `SKILL.md` | Skip that skill; log warning |

**Dependencies**: `pathlib`, `shutil`, `json`, `sys`, `argparse`, `re` (all stdlib). Does NOT import from `compose.py` — the three frontmatter functions (`_parse_frontmatter`, `_body_after_frontmatter`, `_render_frontmatter`) are inlined into `build-platforms.py` to keep it self-contained. This avoids fragile cross-script imports for a 30-line utility.

#### 5. `_write_marketplace_files()` — resolve per-platform source

```python
# Add this mapping alongside MARKETPLACE_PATHS
_MARKETPLACE_PLATFORM = {
    ".github/plugin/marketplace.json": "copilot",
    ".claude-plugin/marketplace.json": "claude",
}

def _write_marketplace_files(registry: dict) -> None:
    mp_info = registry.get("marketplace", {})
    plugins_list = registry.get("plugins", [])

    for mp_path in MARKETPLACE_PATHS:
        platform = _MARKETPLACE_PLATFORM.get(mp_path, "claude")
        output: dict = {}
        if mp_path in _MARKETPLACE_SCHEMA:
            output["$schema"] = _MARKETPLACE_SCHEMA[mp_path]
        output["name"]  = mp_info.get("name", "veiled-market")
        output["owner"] = mp_info.get("owner", {})
        if "metadata" in mp_info:
            output["metadata"] = mp_info["metadata"]

        resolved_plugins = []
        for p in plugins_list:
            entry = {k: v for k, v in p.items() if k != "platform_sources"}
            if "platform_sources" in p:
                platform_source = p["platform_sources"].get(platform)
                if platform_source:
                    entry["source"] = platform_source
            resolved_plugins.append(entry)
        output["plugins"] = resolved_plugins

        os.makedirs(os.path.dirname(mp_path), exist_ok=True)
        with open(mp_path, "w") as f:
            json.dump(output, f, indent=2)
            f.write("\n")
        print(f"Updated {mp_path}")
```

### `schemas/plugins.shema.json` — Modification

**What changes**: Add `platform_sources` as an optional property to the plugin entry definition. Because `plugins.shema.json` references `marketplace.schema.json#/$defs/pluginEntry` (which has no `additionalProperties: false`), the generated marketplace files already tolerate extra fields — this modification is needed only so that schema validation of `sources/plugins.json` itself explicitly allows `platform_sources`.

Since `plugins.shema.json` uses `"$ref": "marketplace.schema.json#/$defs/pluginEntry"` as the item schema for `plugins`, and JSON Schema draft-07 does not allow `additionalProperties` alongside `$ref` without a wrapping `allOf`, the simplest approach is to add `platform_sources` to `marketplace.schema.json`'s `pluginEntry` definition as an optional property. This also makes the field available to anyone validating marketplace files directly.

**Change to `schemas/marketplace.schema.json` — add to `$defs.pluginEntry.properties`:**

```json
"platform_sources": {
  "type": "object",
  "description": "Platform-specific source paths for dual-build plugins. Used by finalize.py to write different source values into each marketplace file. Not written to the generated marketplace.json output files.",
  "additionalProperties": false,
  "properties": {
    "claude":  { "type": "string", "description": "Source path for Claude Code marketplace (use ./ prefix)." },
    "copilot": { "type": "string", "description": "Source path for Copilot CLI marketplace (no ./ prefix)." }
  }
}
```

Note: `platform_sources` is stripped from entries in `_write_marketplace_files()` before the output is written, so it never appears in the generated `.github/plugin/marketplace.json` or `.claude-plugin/marketplace.json`. This keeps those files valid under the existing `marketplace.schema.json`.

## Data Flow

```
Human authors commons/
  └─ plugin.base.json         (shared metadata, version)
  └─ skills/<name>/SKILL.md   (frontmatter: user-invocable: false marks hidden skills)
  └─ agents/<name>.md         (no .agent suffix in commons)
  └─ hooks.json, .mcp.json    (human-authored, identical for both platforms)

finalize.py --step version
  └─ Reads: drafts/<plugin>/commons/plugin.base.json   (fallback: plugin.json)
  └─ Writes: bumped version back to plugin.base.json
  └─ Writes: drafts/<plugin>/CHANGELOG.md

finalize.py --step promote
  └─ shutil.copytree(drafts/<plugin>/ → plugins/<plugin>/)
  └─ Entire tree including commons/ is now at plugins/<plugin>/commons/

finalize.py --step build   [NEW]
  └─ build-platforms.py reads: plugins/<plugin>/commons/
  └─ For each skill, reads SKILL.md frontmatter:
       user-invocable: false → [Claude] keep in frontmatter
                              [Copilot] rename to workflow.md, strip flag
       user-invocable: true or absent → [both] copy as SKILL.md
  └─ Agents: commons/<name>.md → [Claude] claude/<name>.md
                                  [Copilot] copilot/<name>.agent.md
  └─ Writes: plugins/<plugin>/claude/   (complete plugin directory for Claude Code)
  └─ Writes: plugins/<plugin>/copilot/  (complete plugin directory for Copilot CLI)
  └─ Error path: commons/ missing → skip with warning (legacy flat layout)
  └─ Error path: plugin.base.json missing → sys.exit(1)

finalize.py --step marketplace
  └─ Reads: plugins/<plugin>/commons/plugin.base.json
  └─ Builds registry entry with platform_sources:
       { "claude": "./plugins/<plugin>/claude",
         "copilot": "plugins/<plugin>/copilot" }
  └─ Upserts: sources/plugins.json
  └─ _write_marketplace_files() iterates MARKETPLACE_PATHS:
       For .github/plugin/marketplace.json (platform=copilot):
         entry["source"] = platform_sources["copilot"] = "plugins/<plugin>/copilot"
       For .claude-plugin/marketplace.json (platform=claude):
         entry["source"] = platform_sources["claude"] = "./plugins/<plugin>/claude"
       platform_sources key is stripped before writing output
  └─ Writes: .github/plugin/marketplace.json (Copilot CLI marketplace)
  └─ Writes: .claude-plugin/marketplace.json (Claude Code marketplace)

04-finalize-plugin.yml commit step:
  └─ git add "plugins/${plugin}/"   ← covers commons/, claude/, copilot/
  └─ git add ".github/plugin/marketplace.json"
  └─ git add ".claude-plugin/marketplace.json"
  └─ git add "sources/plugins.json"
  └─ git commit -m "chore(finalize): promote ${plugin} to plugins/ [skip ci]"
```

## Critical Notes

**`[skip ci]` is already on the finalize commit** (`git commit -m "chore(finalize): ... [skip ci]"`). The build step runs inside the `finalize` job, which runs on the `finalize/...` branch before the PR is opened. No cascading workflow is triggered.

**Generated files must be committed.** `claude/` and `copilot/` are NOT `.gitignore`d. They are authoritative for the marketplace installer. Do not add them to `.gitignore`.

**`plugins.shema.json` typo is preserved.** All references (in `sources/plugins.json`'s `$schema` pointer, in `ci.yml`'s validation step, in `00-setup-plugins.yml`) use the typo'd filename `plugins.shema.json`. This blueprint makes no changes to that filename or any references to it.

**`_MARKETPLACE_PLATFORM` is the authoritative routing table.** If a third marketplace file is ever added, add its entry here — the rest of `_write_marketplace_files()` will handle it automatically.

**Copilot source path has no `./` prefix; Claude source path requires `./`.** This is a platform-specific behavior of the respective marketplace installers. The mapping is hardcoded in `update_marketplace()`: Claude gets `f"./{plugin_dir}/claude"`, Copilot gets `f"{plugin_dir}/copilot"`.

**The `commons/` directory is never the `source` value in any marketplace file.** It is strictly a build input. Marketplace consumers always point at `claude/` or `copilot/`. This is enforced by `_write_marketplace_files()` stripping `platform_sources` and rewriting `source`.

**Semver bump detection** in `_detect_changelog_diff()` compares `drafts/<plugin>/skills/` vs `plugins/<plugin>/skills/` (immediate child directories). After the migration, both trees gain `commons/`, `claude/`, and `copilot/` as immediate children. The bump detection iterates `("skills", "agents")` subdirectories only, so the new subdirs don't pollute the diff calculation — no change required.
